Requirements

Uses PokeApi to reactively display an index of the first 151 Pokemon

The user can search using either a number which identifies the Pokemon

Information on each Pokemon's stats, types, descriptions, and in-game sprites are displayed upon search through the change of the page and the search bar

The user can go through the index by search, clicking on the nav bar entries, or the arrow navs by the Pokemon images above the data

Since there is only one Pokemon actively displayed at a time we used a drop down to display either the default or shiny in-game sprite

The page meets the controls requirements as well as look uniform and displays the information in an organized manner

The state of the page is seen by which Pokemon is actively displayed, shown by the number and name on top of the page, the middle Pokemon image in the arrow nav above the data and white text on the side nav

Justin Vaughn did the design of the layout and color scheme of the page, as well as developed the navs and Stats section.

Justin Connolly built out the sections of Types, Sprites, and Flavor Text.

The html, css, and javascript were edited by both members of the team collaborativly when we met throughout the process
